Describe and use quick sort and merge sort.


Quick Sort

Choose any element in the list as pivot.

Partition the list in two sub-lists w.r.t pivot.

Left: Elements smaller than or equal to pivot.

Right: Elements greater than pivot.

Each sublist is partioned until the sublists are trival to sort, then everything is recombined.

Break a list down into smaller lists until you get down to a point where you can easily sort them (ie. swap if one element is higher). Then merge these sorted lists, until we finally get a sorted list.



Demonstrate how backtracking can be used to solve problems.


Backtracking

Construct solution from a partial candidate solution.

At each stage:

Evaluate your partial solution (Is it a complete solution?)

Create a list of possible options to further develop the partial solution (avoiding fruitless options).

Try each of the possible options. If no further options to extend solution, backtrack.

Backtracking Requirements

Be able to determine a representation of partial solutions.

keep track of partial solution.

Find a set of next positions.

Keep track of possible next positions.

Be able to undo steps.

Be able to check whether the partial solution is a solution.

Invariants

An invariant is a property that does not change.

So invariants are essentially what make algorithms work. By using one or more properties that don't change, we're able to actually code solutions to problems (Imagine if a sorting algorithm couldn't rely on some part of the list already being sorted.)

Usually in the form of: First k items in list are sorted (i.e., aList[:k] is sorted), or: First k+1 items in list are sorted (i.e., aList[:k+1] is sorted.



Describe and use selection and insert sorts.


Selection Sort

  • Select the smallest number in the list and move it to index 0.
  • Select the 2nd smallest number in the list and move it to index 1.
  • Select the n-th smallest number in the list and move it to index n-1.

Insert Sort

  • Pick 1st number.
  • Pick 2nd number and insert it such that the 2 picked numbers are sorted.
  • Pick the n-th number and insert it such that the n picked cards are sorted.


Apply the greedy approach to solving problems.


A Greedy Algorithm makes a locally optimal choice at each step (In regards to some property.)

Sometimes, but not allways, gives an optimal soltuion.

Prims Algorithm is a greedy algorithm that always gives an optimal solution (The Minimum Spanning Tree).

Coin Denomination Problems only give an optimal solution with some coin denominations.

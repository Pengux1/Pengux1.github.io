Differentiate between variables and how values are stored in Python.


So, Variables point towards a point in memory which stores a specific value. So when you declare a new variable, you're pretty much storing a new value in memory, and then pointing this variable towards it.

This becomes a problem (For Humans) when we set a variable to equal another varaible (Eg. NewVariable = OldVariable). This only does a Shallow Copy, which means that the new variable still points towards the same point in memory as the old one. When the new variable is changed, the old variable will also be changed (and vice-versa). This is mainly a problem when we do something like NewList = OldList * 3, which only performs a shallow copy, meaning when a value is changed in one list, it's changed in all the lists.

To get around this, we need to do a Deep Copy, which creates a new region of memory with the same value as the varaible being copied, which the new variable points towards. We can implement this in Python in many different ways, such as: NewList = list(OldList); NewList = OldList[:], and so on.



Apply the Divide and Conquer strategies including Binary Search to solve problems.


Divide and Conquer

A badass name for a not-so badass strategy. Essentially we Divide a problem into two or more smaller instances, and Conquer these smaller instances by solving them, which are combined to obtain the solution for the problem.

Binary Search

Given a sorted list, we can use a Binary Search to see if an item is contained in the list.

So we take the middle index of the list, and see if the item at this position in the list is equal to the search item. If it is, then easy, we just return the middle index.

If not, we see if the search term is Higher or Lower than this item. If it's lower, we set the end index of the list to be equal to the middle - 1, and if it's higher than the start index of the list is set to equal the middle + 1. A new middle is calculated from the new values of the start/end indexes of the list, and the process is repeated until the list is empty, or the item is found.
